Prototypal Inheritance

Prototypes are objects created for the use of applying a set of attributes to multiple other objects. They act as the parent object, providing default values for the child object that can be overwritten if need be. In the below example, all animals will have legs, but specific animals like "dog" will apply the number of legs on their own.


let animal = {
  hasLegs: true
};

let dog = {
  legCount: 4
};

dog.__proto__ = animal;

console.log(dog.hasLegs);
console.log(dog.legCount);

Functions can be passed through inheritance as well. In the below example, we can see that not only is "dog" gaining the ability to use the method "eat", but the "eat" method also is accessing an attribute only available in "dog" using the keyword 'this'

let animal = {
  hasLegs: true,
  eat() {
    console.log( this.name + " eats!");
  }
};

let dog = {
  name:"Sophie",
  legCount: 4
};

dog.__proto__ = animal;

console.log(dog.hasLegs);
console.log(dog.legCount);

dog.eat();

The keyword 'this' is used to refer to a couple of contexts of your script like the current object or class, but it can also refer to larger scale objects outside of your own scripts functionality. This is due to it refering to the Window object
