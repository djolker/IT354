Week 4 and 5 Topics

Named Functions

Named functions are implementations of a function that includes stating the variables name, accepted variables, and immediately assigning logic to a variable of that functions name

function addFive(x){
 return x + 5;
}
 
console.log(addFive(12));
Function Expressions

This is a method of implementing functions where the function variable is instantiated and then the function value is assigned to it

let addFive = function(x){
 return x + 5;
}
 
console.log(addFive(12));
Arrow Functions

Similar to a function expression, arrow functions allow you to instantiate a variable and assign its logic in one line using the syntax below.

var addFive = (x) => {return x + 5};
 
console.log(addFive(12));
Optional Arguments

Arguments in functions can have default values set for them, so as to make their inclusion optional when called.

function addFive(x = 1) {
 return x + 5;
}
 
console.log(addFive(12));
 
console.log(addFive());

The above code will output 17 when 12 is passed, and 6 when nothing is passed, because the default value of X in this case is 1

Rest Parameters

Sometimes you may want to perform a function on an unknown number of parameters. This is when you would use a rest parameter while defining your function. A rest parameter allows for n number of objects to be passed into the function on call, and treated as an array.

function addTogether(...x) {
 var total = 0;
 for(i of x) {
  total += i;
 }
 return total;
}
 
console.log(addTogether(0, 5, 10));
Nested Functions

In javascript, you have the ability to nest one function inside another by simply defining a new function during the definition of another.

function addTwentyFive(x) {
 function addFive(y) {
  return y + 5;
 }
 
  x = addFive(x);
  x = addFive(x);
  x = addFive(x);
  x = addFive(x);
 x = addFive(x);
 
  return x;
}
 
console.log(addTwentyFive(12));
Scope

Variables have an inside out view of eachother in code, meaning that an object on the innermost layer of a function can see the variables on the outside, but the outside objects can't see in.

var outsideString = "Outside of function";
 
function scopeTest() {
var insideString = "Inside of function";
console.log(outsideString);
console.log(insideString);
}
 
scopeTest();
 
console.log(outsideString);
console.log(insideString);

The last line of the above code will return an error, as it does not have visibility of the variable insideString due to it being inside the function
